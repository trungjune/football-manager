Cách thức hoạt động của Memory Bank

Sơ đồ tổng quan các thành phần và mode chính của hệ thống Memory Bank.
Mỗi mode đại diện cho một giai đoạn trong quy trình phát triển, đảm bảo AI luôn duy trì context và tối ưu hiệu suất.

Thách thức trong phát triển dự án

Trước Memory Bank

  • AI quên context sau mỗi phiên
  • Phải giải thích lại nhiều lần
  • Thiếu quy trình chuẩn hóa
  • Tài liệu không đồng bộ với code

Với Memory Bank

  • AI luôn nhớ bối cảnh dự án
  • Quy trình MODE chuẩn hóa
  • Tài liệu tự động cập nhật
  • Onboarding dễ dàng

Vì sao Cursor Memory Bank phù hợp với dự án ONEDX?

  • 🧩 Quản lý context phức tạp: Dự án ONEDX là multi-portal, nhiều domain, nhiều team – Memory Bank giúp AI luôn nhớ đúng ngữ cảnh từng portal.
  • 🚀 Onboarding nhanh: Thành viên mới chỉ cần đọc Memory Bank là nắm được toàn bộ kiến trúc, quy trình, không cần training thủ công.
  • 🔄 Quy trình chuẩn hóa: MODE rõ ràng, workflow nhất quán, giảm lỗi quy trình, dễ kiểm soát chất lượng.
  • 🤝 Tối ưu teamwork AI + dev: AI và dev cùng làm việc trên nền tảng tài liệu hóa chung, không bị lệch context.
  • 📚 Đảm bảo tài liệu hóa xuyên suốt: Mọi thay đổi, quyết định, tiến độ đều được lưu lại, dễ truy vết và review.
  • 📈 Tăng hiệu suất & chất lượng code: Giảm thời gian lặp lại, AI hỗ trợ đúng trọng tâm, dev tập trung vào logic.
  • 🔧 Dễ mở rộng & bảo trì: Khi dự án phát triển thêm portal, team mới, chỉ cần bổ sung vào Memory Bank, không lo mất context cũ.
  • 🔒 Đảm bảo tuân thủ & kiểm soát: Dễ kiểm tra quy trình, audit, review lại mọi quyết định kỹ thuật.

Cài đặt & cấu trúc Memory Bank

Bước 1: Cài đặt Cursor Editor

  • Tải Cursor Editor (mới nhất)
  • Đăng ký tài khoản, kết nối API (nếu cần, GPT-4 khuyến nghị)

Bước 2: Tạo cấu trúc Memory Bank

  • Clone dự án ONEDX đã có sẵn memory-bank/
  • Kiểm tra đủ các file cốt lõi:
memory-bank/
├── projectbrief.md
├── productContext.md
├── systemPatterns.md
├── techContext.md
├── activeContext.md
├── progress.md
├── tasks.md
├── SETUP_GUIDE.md
├── USAGE_GUIDE.md
├── userRules.md
└── custom_modes/
    ├── van_instructions.md
    ├── plan_instructions.md
    ├── creative_instructions.md
    ├── implement_instructions.md
    └── reflect_archive_instructions.md

Thiết lập Custom Modes & Quy trình sử dụng

Bước 3: Thiết lập Custom Modes

  • Settings → Features → Bật Allow the creation of custom modes (BETA)
  • Khởi động lại Cursor nếu được nhắc
  • Tạo 6 mode: VAN, PLAN, CREATIVE, IMPLEMENT, REFLECT, ARCHIVE
  • Dán hướng dẫn từ custom_modes/ vào Advanced options cho từng mode

Bước 4: Thiết lập Rules

  • Settings → Rules
  • Dán userRules.md vào Custom instructions
  • Lưu & khởi động lại Cursor

Quy trình sử dụng

  • Bắt đầu với VAN mode → Gõ VAN
  • Làm theo: VAN → PLAN → CREATIVE → IMPLEMENT → REFLECT → ARCHIVE (tùy task)
  • Cập nhật Memory Bank sau mỗi thay đổi lớn

Lưu ý & xử lý sự cố

  • Kiểm tra đúng hướng dẫn cho từng mode
  • Nếu lỗi, xem SETUP_GUIDE.md & USAGE_GUIDE.md

Các MODE chính

Mode Mô tả Vai trò
🔍 VAN Verification & Navigation Điểm khởi đầu, phân tích dự án, xác định độ phức tạp
📋 PLAN Lập kế hoạch Phân tích yêu cầu, lập kế hoạch từng bước
🎨 CREATIVE Thiết kế Thiết kế giải pháp, đề xuất pattern/kiến trúc
⚒️ IMPLEMENT Thực thi Viết code, thực hiện giải pháp
🔍 REFLECT Đánh giá Review code, test và verify
📚 ARCHIVE Lưu trữ Cập nhật tài liệu, đóng task

PLAN MODE

↓
CREATIVE MODE

↓
IMPLEMENT MODE

↓

↓
ARCHIVE MODE
Cập nhật tài liệu, tổng kết, đóng task, cập nhật Memory Bank
